3. Modbus/TCP — 4가지 Function Code

Modbus는 PLC 메모리를 4가지 영역으로 나누고, 각 영역마다 읽기/쓰기 명령(Function Code)을 정의합니다.

핵심 원리 — 두 가지 질문

1. 비트인가, 워드인가?

ON/OFF 신호(비트) vs 온도·압력·카운터(워드)

2. 읽기만 가능한가, 쓰기도 되는가?

센서 측정값(RO) vs 설정값·지령(R/W)

FC 영역명 데이터 방향 LS XGT 매핑 주요 용도
01 Coil 비트 R/W %QX (디지털 출력) 밸브, 솔레노이드, 표시등 ON/OFF
02 Discrete Input 비트 RO %IX (디지털 입력) 센서, 리밋 스위치 감지
03 Input Register 워드 RO %IW (아날로그 입력) 온도계, 압력계 측정값
04 Holding Register 워드 R/W %MW / D 영역 설정값, 파라미터, 카운터 — 현장의 90%
암기 팁 짝수(02·04) = 읽기 전용(RO) / 홀수(01·03) = 양방향(R/W)
번호 작은 쪽(01·02) = 비트 / 번호 큰 쪽(03·04) = 워드
현장 실무의 90% 이상은 FC04 Holding Register — PLC 프로그래머들이 통신용 데이터를 D 영역(%MW)에 모아두기 때문.

4. 바이트·워드 순서 — 실수 방지 가이드

왜 중요한가?

온도 센서가 32비트 부동소수점 1234.5를 보냄: 0x44 9A 50 00

설정바이트 조합float 해석 결과
BIG_ENDIAN + HIGH_WORD_FIRST (ABCD)44 9A 50 001234.5 ✓ 정상
LITTLE_ENDIAN + LOW_WORD_FIRST (DCBA)00 50 9A 447.39e-39 (오류)
BIG_ENDIAN + LOW_WORD_FIRST (BADC)9A 44 00 50-1.0e-22 (오류)
LITTLE_ENDIAN + HIGH_WORD_FIRST (CDAB)50 00 44 9A8.59e9 (오류)
설정 1개가 틀리면 값이 완전히 망가집니다 자바에서 해석할 때는 ByteBuffer.order(ByteOrder.BIG_ENDIAN)과 같은 명시적 설정이 필수입니다.

주소 베이스 — 0-base vs 1-base

PLC 매뉴얼 표기:    Holding Register 40001 ~ 49999  (1-base)
실제 TCP 패킷 전송: 0x0000 ~ 0x270E              (0-base, -40001 보정)

예: 매뉴얼 "40007을 읽어라" → 패킷에는 6 전송 (40007 - 40001 = 6)
라이브러리마다 기준이 다름 — 반드시 확인 Modbus4j: 0-base 직접 입력 / 일부 라이브러리: 내부에서 자동 보정
처음 통신 테스트할 때 알려진 값(예: 가동 표시등)으로 ±1 모두 시도해 결정하세요.

5. 실무 적용 순서와 검증 방법

  1. [PLC 담당자] 매핑표 수령
    XG-PD에서 D 영역 ↔ Modbus 주소 매핑 확인. 데이터 타입·스케일링·단위 포함 요청.
  2. [AA0040] PLC 연결 정보 등록
    IP, 포트(502), Unit ID(1), 주소 베이스(0), 바이트 순서(BIG_ENDIAN), 워드 순서(HIGH_WORD_FIRST) 입력.
  3. [AA0040] 연결 테스트 버튼으로 TCP 502 접속 확인
    실패 시: 방화벽 502/TCP 개방 여부, PLC 고정 IP, XG-PD Modbus Server 활성화 확인.
  4. [AA0030] 수집 항목 등록
    메모리 영역(FC04_HOLDING_REG), PLC 주소, 레지스터 수, 데이터 타입 입력.
  5. [검증] 알려진 값으로 첫 검증
    호기 가동 표시등 ON/OFF 시키며 비트값 변화 확인. 값이 예상과 다르면 바이트/워드 순서 역추적.
  6. [운영] 600개 노드 전체 검증 후 본격 폴링

6. PLC 기초 — 자바 개발자 관점

PLC(Programmable Logic Controller)는 공장 설비를 제어하는 산업용 컴퓨터입니다. 자바 개발자 관점에서는 "외부와 통신 가능한 임베디드 컴퓨터", 또는 "SQL 대신 Modbus로 접근하는 특수한 데이터베이스"로 이해하면 됩니다.

수집 (READ)

PLC에서 측정값·카운터·상태를 주기적으로 읽어 DB에 저장하거나 모니터링 화면에 표시.

제어 (WRITE)

MES/SCADA에서 레시피·설정값·지령을 PLC에 내려보내 설비 동작 변경.

분석 및 대시보드

수집된 시계열 데이터로 가동률·생산량·품질 분석, 웹 UI 시각화.

7. PLC 메모리의 두 단위 — 비트와 워드

비트(Bit) — ON/OFF 신호

  • 센서 감지 여부 (감지=1, 미감지=0)
  • 모터 가동 여부 (가동=1, 정지=0)
  • 도어 열림/닫힘
  • 비상정지 버튼 상태

1비트면 충분. 워드를 쓰면 메모리 낭비.

워드(Word) — 수치 값 (16비트)

  • 온도: 25.7℃ → 정수 257로 저장
  • 무게: 1,857.9 kg → 정수 18579
  • 생산 카운터: 1,234개
  • 인버터 속도: 1,800 rpm

16비트(0~65535) 기본. 더 큰 값은 32비트(더블워드).

자바 데이터 타입 대응표

PLC 단위크기자바 타입표현 범위
bit1 bitbooleantrue / false
word (INT16, 부호 있음)16 bitshort-32,768 ~ 32,767
word (UINT16, 부호 없음)16 bitint (양수만)0 ~ 65,535
double word (INT32)32 bitint약 ±21억
float (REAL, FLOAT32)32 bitfloatIEEE 754
double (LREAL, FLOAT64)64 bitdoubleIEEE 754
주의 — 부호 처리 PLC는 기본적으로 부호 있는 16비트 정수(short)를 가장 많이 씁니다. 음수 처리, UINT16 vs INT16 구분이 잘못되면 65535가 -1로 보이거나 그 반대가 됩니다.

8. PLC 메모리 영역과 표기법

3가지 영역 구분

영역이름설명비유
외부 입출력 %IX, %IW, %QX, %QW 실제 센서·액추에이터와 전선으로 연결된 영역 회사 출입문 · 외부 연락처
내부 %MX, %MW (D, M 등) PLC 프로그램 내부에서만 사용하는 임시 변수 직원 메모지 · 계산기
시스템 %SW PLC가 자기 상태를 자동으로 기록 (사용자가 만들지 않음) 출퇴근 기록부 · 건강검진 기록

% 표기법 해석 — IEC 61131-3 국제 표준

표기분해풀어쓰기한국어 의미예시 용도
%IXI + XInput Bit입력 비트근접센서 감지 신호 (RO)
%QXQ + XOutput Bit출력 비트솔레노이드 ON/OFF (R/W)
%MXM + XMemory Bit내부 비트가동조건 충족 플래그 (R/W)
%IWI + WInput Word입력 워드온도 측정값 (RO)
%QWQ + WOutput Word출력 워드인버터 속도지령 (R/W)
%MWM + WMemory Word내부 워드설정 목표값, 누적 카운터 (R/W)
%SWS + WSystem Word시스템 워드PLC 스캔타임, 에러코드 (RO)
Q가 Output인 이유 영문자 O는 숫자 0과 혼동되기 쉬워 일부러 피했습니다. O 바로 다음 글자인 Q를 Output에 사용합니다.
시스템 워드(%SW) 쓰기 금지 %SW는 PLC가 자체 관리하는 영역. 외부에서 쓰면 PLC 오동작. 수집(READ)만 허용.

9. LS Electric XGT 메모리 영역 매핑표

LS Electric XGT 시리즈의 메모리 영역과 Modbus Function Code, IEC 61131-3 표준 표기의 대응 관계입니다.

Modbus FC LS XGT 영역 IEC 표준 비트/워드 방향 주요 용도

FC02 Discrete Input P (디지털 입력) %IX 비트 RO 근접센서, 리밋 스위치

— M (내부 비트) %MX 비트 R/W 프로그램 내부 플래그
— F (시스템 워드) %SW 워드 RO PLC 상태, 에러코드 — 쓰기 금지

10. IP와 주소만으로 통신이 안 되는 이유 — 13개 항목

"PLC IP 192.168.10.10과 메모리 주소 5110을 안다"는 정보만으로는 데이터를 읽을 수도, 해석할 수도 없습니다. 최소 8가지 추가 정보가 필요하고, 실제 운영에는 13개 항목이 필요합니다.

#항목예시없으면 발생하는 문제
1IP 주소192.168.10.10TCP 연결 목적지 불명
2포트 번호502TCP 연결 자체 실패
3유닛 ID1게이트웨이 뒤 여러 PLC 구분 불가
4Function Code04어떤 메모리 영역인지 불명확
5주소 베이스0-base1개 오프셋 차이로 엉뚱한 값 읽힘
6데이터 타입FLOAT3216/32비트 구분, 부호 유무 모름
7바이트 순서BIG_ENDIANINT32·FLOAT 값이 완전히 다르게 해석
8워드 순서HIGH_WORD_FIRST32비트 값이 수천만 배 차이
9레지스터 수2FLOAT32를 1개만 읽으면 절반만 읽음
10스케일링×0.1원시값 257이 25.7℃인지 257℃인지 모름
11오프셋0값 보정 누락
12단위℃의미 없는 숫자만 저장됨
13설명호기1 가열로 온도어떤 데이터인지 맥락 불명
 
# 13개 항목이 모인 데이터 카탈로그 예시 (YAML)
- node_id: PLC_6F_01_TEMP

  word_order: high_word_first     # 10. 워드 순서
  scaling: 0.1                    # 11. 스케일링 (원시 257 → 25.7)
  unit: degC                      # 12. 단위
  description: 6층 1호기 가열로 온도  # 13. 설명

11. 코드 설계 — READ와 WRITE의 분리

구분READ (수집)WRITE (제어)
대상 영역모든 영역 (RO + R/W)R/W 영역만 — %QX, %MW, D 영역
주기주기적 폴링 (자동)요청 발생 시점 (이벤트)
권한일반 권한도 가능관리자/운영자 권한 필수
로그감사로그 불필요SystemLog 필수 (누가/언제/어떤 값)
실패 영향모니터링 갭 (한 주기 누락)설비 오동작·안전사고 가능
 
// WRITE (제어) — 권한 필수, 감사로그 필수
@Service
public class PlcWriteService {

    @RequirePermission(action = "update")
    public void writeNode(String nodeId, Object value, User user) {
        PlcNode node = catalogRepo.findById(nodeId);

        // RO 영역 쓰기 차단
        if (node.getDirection() == Direction.READ_ONLY) {
            throw new IllegalOperationException("RO 영역 쓰기 금지: " + nodeId);
        }

        performWrite(node, value);

        // 감사로그 필수
        systemLogService.log(user, "PLC_WRITE", nodeId, value, Instant.now());
    }
}
절대 규칙 RO 영역(%IX, %IW, %SW)에 WRITE 시도는 코드 레벨에서 즉시 예외 발생으로 막아야 합니다. PLC가 알아서 거부해줄 거라 믿으면 안 됩니다.

12. 자바 개발자 체크리스트와 흔한 함정

설계 단계

  • ☐ PLC 담당자로부터 매핑표(Memory Map) 수령 (13개 항목 전부 포함 확인)
  • ☐ 통신 프로토콜 결정 (이 프로젝트: Modbus/TCP)
  • ☐ 폴링 주기 정의 (T1 실시간: 100ms, T2 비실시간: 1초, T3 비필수: 10초)
  • ☐ 동시 접속 한도 확인 (XGL-EFMTB: 최대 16 클라이언트)

구현 단계

  • ☐ Modbus 라이브러리 선택 (Modbus4j, j2mod 등)
  • ☐ TCP 연결 풀(Connection Pool) 구성 — 매 폴링마다 connect/close 금지
  • ☐ 연속 주소 묶어 읽기 (Bulk Read) 구현 — 최대 125 워드/요청
  • ☐ 데이터 타입 변환 로직 (INT16, INT32, FLOAT32, 바이트 순서 명시적 설정)
  • ☐ READ/WRITE 서비스 분리
  • ☐ RO 영역 쓰기 차단 가드 구현

운영 단계

  • ☐ 방화벽 502/TCP 개방 확인
  • ☐ XG-PD에서 Modbus/TCP Server 활성화 + 다운로드 확인
  • ☐ 통신 실패율·재시도 카운터 메트릭
  • ☐ 보안: Modbus/TCP는 기본 인증 없음 → 네트워크 격리 필수 (VLAN, 방화벽)

흔히 만나는 함정

증상의심 원인확인 방법

음수가 큰 양수로 보임 INT16 vs UINT16 혼동 자바 short(signed)로 캐스팅 후 비교
