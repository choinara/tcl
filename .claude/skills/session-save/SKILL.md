# /session-save — 세션 컨텍스트 저장

지금까지의 작업 내용을 바탕으로 `myclaudemd/CLAUDE.md`를 갱신해라.
루트 `CLAUDE.md`는 **결정사항/아키텍처/주의사항**만 담고 있으므로, 세션 이력은 `myclaudemd/CLAUDE.md`에 기록한다.

## 갱신 대상

### 1차: `myclaudemd/CLAUDE.md` (세션 이력)

- 기존 파일을 **먼저 읽고**, 섹션별로 병합해라. **전체 교체 금지.**
- "현재 작업" 섹션: 이번 세션 내용으로 교체
- "이전 작업 기록" 섹션: 기존 유지 + 이번 세션 완료분 append (날짜 포함)
  - **중복 방지**: append 전 "이전 작업 기록"에 동일 날짜+세션명의 `### YYYY-MM-DD` 항목이 이미 있으면 추가하지 않는다. "현재 작업" 섹션의 날짜·세션명과 기존 항목을 대조해 판단

### 2차: 루트 `CLAUDE.md` (프로젝트 컨텍스트) — 해당 시에만

- 새로운 **결정사항**(아키텍처, 네이밍, 패턴 변경)이 확정된 경우에만 "결정사항" 섹션에 append
- "완료된 작업" 섹션에 이번 세션 완료분 append (날짜 포함)
- "다음 할 일" 섹션에서 완료 항목 제거 + 신규 추가
- 세션 이력/작업 상세는 루트 CLAUDE.md에 쓰지 않는다

### 3차: 오답노트 + 메모리

- `memory/` 폴더: 이번 세션에서 새로 확인된 프로젝트 결정사항이나 피드백이 있으면 메모리 파일도 갱신

- **오답노트** (`/Users/hschoi/Documents/Git/CHS/docs/오답노트.md`): 이번 세션에서 발생한 오류/이슈가 있으면 아래 순서로 추가

  **이슈가 없는 세션은 오답노트를 수정하지 않는다.**

  #### 오답노트 작성 순서 (3단계 필수)

  **Step 1. 파일 전체 구조 파악**
  - 파일을 먼저 읽어 현재 최고 이슈 번호(N)와 목차 카테고리(A~H) 확인
  - 기존 내용에 **append** (전체 교체 절대 금지)

  **Step 2. 이슈 목록 테이블에 행 추가** (`## 이슈 목록` 섹션 하단)
  ```
  | N+1 | YYYY-MM-DD | 제목 (한 줄 요약) | 범주명 |
  ```
  - 범주명: 기존 동일 유형이 있으면 동일 범주명 재사용

  **Step 3. 상세 기록 추가** (`## 상세 기록` 섹션 하단)
  ```markdown
  ### #N+1. 프로젝트명/제목

  **날짜**: YYYY-MM-DD
  **메뉴**: 메뉴코드 또는 작업명
  **범주**: 범주명

  #### 1. 발생분야

  (백엔드/프론트엔드/DB 등 어느 영역에서 발생했는지)

  #### 2. 발생원인

  (근본 원인 — 코드 스니펫 포함)

  #### 3. 최종 해결방법

  (해결 코드 + 예방법)
  ```

  **Step 4. 목차 업데이트** (`## 목차 (카테고리별)` 섹션) — **반드시 수행**
  - 새 이슈가 속하는 카테고리(A~H)를 찾아 해당 항목에 `**#N+1**` 추가
  - 기존 카테고리에 맞지 않는 경우: 새 카테고리 항목 추가 (예: `### I. 새카테고리명`)
  - 목차 항목 형식: `- 이슈 한 줄 설명: **#N+1**`

## CLAUDE.md 구조

```markdown
# 프로젝트 컨텍스트

## 현재 작업 (이번 세션에서 진행한 것)
- 구체적으로 어떤 파일을 어떻게 수정했는지 작성

## 결정사항 (앞으로 바꾸지 말 것)
- 아키텍처, 네이밍 규칙, 에러 처리 방식 등

## 완료된 작업 (날짜 포함해서 누적)
- YYYY-MM-DD: 완료 내용

## 다음 할 일 (우선순위 순서로)
- [ ] 1순위 작업
- [ ] 2순위 작업

## 현재 파일 구조 중 중요한 것
- 수정한 파일 경로와 역할 요약

## 주의사항 (건드리면 안 되는 것)
- 컨테이너 이름, 환경변수, 의존성 등
```

## 추가 갱신 대상

- `memory/` 폴더: 이번 세션에서 새로 확인된 프로젝트 결정사항이나 피드백이 있으면 메모리 파일도 갱신

## 완료 후

"세션 컨텍스트 저장 완료. 다음 세션에서 `/session-load`으로 복기할 수 있습니다." 출력.
